## 四、任务架构（13 个 FreeRTOS 任务）

```
                         main()
                           │
                           ▼
                     app_task_init (优先级 6)
                     ┌─────────────────────┐
                     │ 创建互斥锁 × 3       │
                     │ 创建事件标志组       │
                     │ 创建消息队列 × 2     │
                     │ 创建计数信号量       │
                     │ 初始化全部硬件 + LVGL│
                     │ 遍历 task_tab[] 批量  │
                     │   创建 11 个任务     │
                     │ 启动 1s 软件定时器   │
                     │ 删除自身             │
                     └─────────────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
   ┌────▼────┐      ┌──────▼──────┐     ┌────▼────┐
   │ LVGL 刷新│      │ 传感器采集   │     │ 控制/通信│
   │ (Prio 6) │      │ (Prio 5~7)  │     │ (Prio 5) │
   └─────────┘      └─────────────┘     └─────────┘
```

### 任务详情

| 任务 | 优先级 | 栈（字） | 职责 |
|:-----|:---:|:---:|:-----|
| `app_task_lvgl` | 6 | 2048 | 循环调用 `lv_task_handler()`，5ms 周期，持 LVGL 互斥锁 |
| `app_task_max30102` | 7 | 2048 | 心率血氧采样（500 点滑动窗口），刷新图表控件（**最高优先级，启动后挂起**） |
| `app_task_rtc` | 5 | 512 | 等事件标志 → 读 RTC → 刷新时间/日期/星期 |
| `app_task_dht11` | 6 | 512 | 每 5 秒读温湿度 → 刷新进度弧 |
| `app_task_motor` | 5 | 128 | 消费马达队列，GPIO 控制震动，完成后给信号量 |
| `app_task_rtc_alarm` | 5 | 128 | 等闹钟事件 → 向马达队列发 5 次震动（每次间隔 1s） |
| `app_task_ble` | 5 | 512 | 解析蓝牙队列中的字符串命令（TIME SET / DATE SET / ALARM SET） |
| `app_task_icon` | 5 | 128 | 每 100ms 刷新顶部状态图标（闹钟/久坐/蓝牙连接） |
| `app_task_mpu6050` | 5 | 512 | 抬腕亮屏检测（roll > 15°）+ 每 5 秒读 DMP 计步数 |
| `app_task_sedentary_remind` | 5 | 512 | 久坐检测（启动挂起，步数增量 < 5 则震动提醒） |
| `app_task_step_clear` | 5 | 128 | 每天 >23 点清零计步 |
| `app_task_istouch_lcd` | 6 | 128 | 触摸唤醒屏幕 + 重置无操作计数 |

## 五、IPC 通信机制

```
                         事件标志组 (RTC ISR → Task)
                         ┌──────────────────────┐
       USART3 ISR        │  RTC_ALARM_IRQHandler│
       (bt24.c:98)       │  RTC_WKUP_IRQHandler │
            │            └──────┬───────────────┘
     xQueueSendFromISR          │ xEventGroupSetBitsFromISR
            │                   │
            ▼                   ▼
      g_queue_bt24_handle   g_event_group_handle
      (消息队列 10×32)      (事件标志组)
            │                   │
            ▼                   ├─ EVENT_GROUPS_RTC_WAKEUP (0x01) → app_task_rtc
      app_task_ble             └─ EVENT_GROUPS_RTC_ALARM  (0x02) → app_task_rtc_alarm
     (蓝牙命令解析)
            │
            ▼
      RTC_SetTime / RTC_SetDate / RTC_SetAlarm


      ┌──────────────┐    xQueueSend     ┌──────────────┐    GPIO     ┌──────┐
      │ 闹钟/久坐提醒 │ ──────────────→  │ app_task_motor │ ─────────→  │ 马达 │
      │ 等多提醒源    │                  │  (消费者)     │            └──────┘
      └──────────────┘                   └──────┬───────┘
                                                │ xSemaphoreGive
                                                ▼
                                        g_sem_motor_handle
                                      (计数信号量，串行化多震动)

      ┌──────────────┐
      │ 多任务写屏    │ ── g_mutex_lvgl_handle ──→ LVGL 控件线程安全
      └──────────────┘       (互斥锁)

      printf / sprintf ── g_mutex_printf_handle / g_mutex_sprintf_handle (互斥锁)
```

| IPC 类型 | 对象名 | 用途 |
|:---------|:-------|:-----|
| **互斥锁** (×3) | `g_mutex_printf_handle` | 保护 printf 调用的重入安全 |
| | `g_mutex_sprintf_handle` | 保护 sprintf 调用的重入安全 |
| | `g_mutex_lvgl_handle` | 保护 LVGL 控件操作的非线程安全访问 |
| **事件标志组** (×1) | `g_event_group_handle` | RTC 唤醒/闹钟 ISR 通知对应任务 |
| **消息队列** (×2) | `g_queue_motor_handle` (5×8) | 马达震动命令 |
| | `g_queue_bt24_handle` (10×32) | 蓝牙接收数据帧（ISR → Task） |
| **计数信号量** (×1) | `g_sem_motor_handle` | 马达执行完成回执，串行化多次震动 |

## 十、技术要点总结

1. **LVGL 线程安全**：所有写屏操作（12 个任务）均通过 `g_mutex_lvgl_handle` 互斥锁进行保护，避免多任务并发刷新导致的崩溃和显示错乱。

2. **MAX30102 滑动窗口算法**：500 点 FIFO 数据窗口，先采满 500 点做基线计算，之后每轮弃前 100、补 100 新样、续算 400 历史，保持连续心率/血氧输出。

3. **蓝牙不定长收帧**：利用 USART3 的 **IDLE 空闲中断** 判定帧边界，避免超时计时器开销，ISR 中通过 `xQueueSendFromISR` 将整帧推入队列，与任务层解耦。

4. **马达串行化**：多个提醒源（闹钟、久坐）通过队列发送震动命令，马达任务执行后通过计数信号量回执，保证多次震动按序执行、互不抢占。

5. **低功耗策略**：软件定时器累计无操作时长 → 自动熄屏 + `__WFI()` 睡眠，支持触摸和抬腕两种唤醒方式。

6. **MPU6050 DMP**：使用 InvenSense eMPL 库进行硬件姿态解算和计步，避免 CPU 做软件姿态融合。
